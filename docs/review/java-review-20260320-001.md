## 一、设计模式分析

### 1.1 使用合理的设计模式

| 模式 | 位置 | 评价 |
|------|------|------|
| **Builder 模式** | `WebSocketConfig`, `TcpConfig`, `JwschConfig` 等配置类 | ✅ 优秀。配置类字段较多，Builder 模式使代码清晰易读 |
| **策略模式** | `LoadBalance` 接口及实现（`RoundRobinLoadBalance`, `RandomLoadBalance`, `ConsistentHashLoadBalance`） | ✅ 优秀。负载均衡策略可插拔，符合开闭原则 |

| **模板方法模式** | `AbstractNettyServer` | ✅ 合理。抽取 Netty 服务器公共逻辑 |
| **单例模式** | `SharedEventLoopManager` | ✅ 合理。共享 EventLoop 资源管理 |
| **空对象模式** | `NoOpServerMetrics`, `NoOpTopicStatsManager` | ✅ 优秀。消除 null 检查，代码更简洁 |
